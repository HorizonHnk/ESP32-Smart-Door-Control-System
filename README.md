## üìç Pin Configuration

### ESP32 Pin Mapping

```cpp
// I2C Communication (LCD)
SDA = GPIO 21  // I2C Data
SCL = GPIO 22  // I2C Clock

// Sensors
ULTRASONIC_TRIG_PIN = GPIO 19  // Ultrasonic trigger
ULTRASONIC_ECHO_PIN = GPIO 18  // Ultrasonic echo
DHT_PIN = GPIO 25              // DHT22 data (needs 4.7k-10kŒ© pull-up)
IR_RECEIVER_PIN = GPIO 14      // IR receiver

// Actuators & Outputs
SERVO_PIN = GPIO 26            // Servo motor PWM
RELAY_PIN = GPIO 27            // Relay module control
BUZZER_PIN = GPIO 15           // Active buzzer
LED_STATUS_PIN = GPIO 5        // Status LED
TOGGLE_LED_PIN = GPIO 13       // Toggle LED
FAN_MOTOR_PIN = GPIO 32        // DC Fan PWM

// Input Buttons
EMERGENCY_EXIT_PIN = GPIO 34   // Emergency button (input-only, external pull-up)
TOGGLE_BUTTON_PIN = GPIO 35    // LED toggle button (external pull-up)
FAN_BUTTON_PIN = GPIO 23       // Fan toggle button (external pull-up)
```

### Important Pin Notes

> **‚ö†Ô∏è GPIO 34-39**: Input-only pins, require external pull-up resistors (10kŒ©)
> 
> **‚ö†Ô∏è GPIO 25**: ADC2 pin, may conflict with WiFi if using ADC
> 
> **‚ö†Ô∏è GPIO 5**: Strapping pin, ensure proper startup state
> 
> **‚ö†Ô∏è DHT22**: Requires external pull-up resistor (4.7kŒ© - 10kŒ©) on data pin

### LCD I2C Address
- Default: `0x27`
- Alternative: `0x3F` (if 0x27 doesn't work)
- Use I2C scanner sketch to find your LCD address

## üöÄ Installation

### Step 1: Hardware Assembly

1. **Connect I2C LCD**:
   ```
   LCD VCC ‚Üí ESP32 5V
   LCD GND ‚Üí ESP32 GND
   LCD SDA ‚Üí GPIO 21
   LCD SCL ‚Üí GPIO 22
   ```

2. **Connect DHT22**:
   ```
   DHT22 VCC ‚Üí ESP32 3.3V
   DHT22 GND ‚Üí ESP32 GND
   DHT22 DATA ‚Üí GPIO 25 (with 4.7kŒ© pull-up to 3.3V)
   ```

3. **Connect HC-SR04**:
   ```
   TRIG ‚Üí GPIO 19
   ECHO ‚Üí GPIO 18
   VCC ‚Üí 5V
   GND ‚Üí GND
   ```

4. **Connect Servo Motor**:
   ```
   Signal ‚Üí GPIO 26
   VCC ‚Üí External 5V (recommended)
   GND ‚Üí Common GND
   ```

5. **Connect remaining components** following the [Pin Configuration](#-pin-configuration) table.

### Step 2: Software Installation

1. **Clone Repository**:
   ```bash
   git clone https://github.com/yourusername/esp32-smart-door-control.git
   cd esp32-smart-door-control
   ```

2. **Open in Arduino IDE**:
   - Open `esp32_door_control.ino`

3. **Configure WiFi Credentials**:
   ```cpp
   const char* ssid = "YOUR_WIFI_SSID";
   const char* password = "YOUR_WIFI_PASSWORD";
   ```

4. **Configure API Keys** (optional, can be set via web interface later):
   ```cpp
   String GEMINI_API_KEY = "YOUR_GEMINI_API_KEY";
   String FORMSPREE_ENDPOINT = "https://formspree.io/f/YOUR_FORM_ID";
   ```

5. **Select Board & Port**:
   - **Board**: ESP32 Dev Module
   - **Upload Speed**: 115200
   - **Port**: Select your ESP32 COM port

6. **Upload Code**:
   - Click **Upload** button
   - Monitor Serial output (115200 baud) for IP address

### Step 3: Initial Setup

1. **Find IP Address**:
   - Check Serial Monitor for: `IP Address: 192.168.x.x`
   - Or check your router's connected devices

2. **Access Web Dashboard**:
   - Open browser: `http://192.168.x.x`

3. **Configure Settings**:
   - Navigate to **Settings** tab
   - Update thresholds, servo angles, API keys
   - Click **Save All Settings**

## ‚ö° System Behavior

### Obstacle Detection Mode
```
Distance < Threshold ‚Üí Relay ON (Lights ON)
Distance > Threshold ‚Üí Relay OFF (Lights OFF)
```

### Door Control Modes

#### 1. Manual/Remote Opening
```
Trigger (Button/IR/Web) ‚Üí Door opens 90¬∞
‚Üí Buzzer beeps (400ms on, 200ms off)
‚Üí Status LED flashes (250ms interval)
‚Üí Auto-close after 10 seconds (configurable)
‚Üí Door closes to 0¬∞
```

#### 2. Automatic Ventilation
```
Temp ‚â• Threshold OR Humidity ‚â• Threshold
‚Üí Door opens 90¬∞ (VENTILATION mode)
‚Üí Fan activates automatically
‚Üí Buzzer beeps (600ms on, 400ms off)
‚Üí Email alert sent (5-min cooldown)
‚Üí Door stays open until BOTH conditions normalize
‚Üí No auto-close timer
```

#### 3. Multiple Alerts (Urgent)
```
Door Open AND Temp/Humidity Alert
‚Üí Rapid buzzer beeps (200ms on/off)
‚Üí Visual indicators active
‚Üí Continuous monitoring
```

### LCD Display Layout

```
Line 1: == Door Control ==
Line 2: Status: CLOSED/OPEN (Xs)
Line 3: Alert/Obstacle/LED+Fan Status
Line 4: T:25.5C H:65% L:OFF
```

### Buzzer Patterns

| Pattern | Timing | Condition |
|---------|--------|-----------|
| **Manual Door** | 400ms ON / 200ms OFF | Door manually opened |
| **Temp Alert** | 600ms ON / 400ms OFF | Temperature/humidity alert only |
| **Multiple Alerts** | 200ms ON / 200ms OFF | Door open + Temp alert (urgent!) |
| **Manual Test** | 100ms ON / 100ms OFF | Buzzer toggle via web/remote |

## üîç Troubleshooting

### WiFi Connection Issues

**Problem**: ESP32 not connecting to WiFi

**Solutions**:
- ‚úÖ Verify SSID and password are correct
- ‚úÖ Check 2.4GHz WiFi (ESP32 doesn't support 5GHz)
- ‚úÖ Ensure router is within range
- ‚úÖ Check Serial Monitor for connection attempts
- ‚úÖ Try different WiFi channel on router

### LCD Display Issues

**Problem**: LCD shows garbage characters or nothing

**Solutions**:
- ‚úÖ Verify I2C address (try 0x27 or 0x3F)
- ‚úÖ Check wiring: SDA‚ÜíGPIO21, SCL‚ÜíGPIO22
- ‚úÖ Run I2C scanner sketch to find address
- ‚úÖ Adjust contrast potentiometer on LCD backpack
- ‚úÖ Check 5V power supply is stable

### DHT22 Reading NaN

**Problem**: Temperature/Humidity show "nan"

**Solutions**:
- ‚úÖ Add 4.7kŒ© - 10kŒ© pull-up resistor to DATA pin
- ‚úÖ Check sensor wiring and orientation
- ‚úÖ Wait 2-3 seconds after power-on for sensor initialization
- ‚úÖ Verify DHT_PIN matches your wiring
- ‚úÖ Try different DHT22 sensor (could be defective)

### IR Remote Not Working

**Problem**: IR remote commands not recognized

**Solutions**:
- ‚úÖ Check IR receiver polarity (OUT, GND, VCC)
- ‚úÖ Verify IR_RECEIVER_PIN (GPIO 14)
- ‚úÖ Point remote directly at receiver (3-5 meters range)
- ‚úÖ Check Serial Monitor for IR codes
- ‚úÖ Update IR button codes in code to match your remote
- ‚úÖ Replace IR receiver if damaged

### Servo Not Moving

**Problem**: Servo doesn't respond to commands

**Solutions**:
- ‚úÖ Use external 5V/2A power for servo (high current draw)
- ‚úÖ Common ground between ESP32 and servo power
- ‚úÖ Check SERVO_PIN (GPIO 26) connection
- ‚úÖ Verify servo angles (0-180¬∞ range)
- ‚úÖ Test with simple servo sweep sketch
- ‚úÖ Check Serial Monitor for servo commands

### Web Dashboard Not Loading

**Problem**: Cannot access web interface

**Solutions**:
- ‚úÖ Check Serial Monitor for IP address
- ‚úÖ Verify ESP32 and computer on same network
- ‚úÖ Try accessing via hostname if mDNS enabled
- ‚úÖ Disable browser extensions (AdBlock, etc.)
- ‚úÖ Clear browser cache and cookies
- ‚úÖ Check firewall settings

### AI Chatbot Not Responding

**Problem**: Chatbot shows error message

**Solutions**:
- ‚úÖ Verify Gemini API key is correct and active
- ‚úÖ Check API key quota hasn't exceeded
- ‚úÖ Ensure internet connectivity is stable
- ‚úÖ Check browser console for specific errors
- ‚úÖ Try regenerating API key from Google AI Studio

### Buzzer Not Sounding

**Problem**: Buzzer doesn't beep

**Solutions**:
- ‚úÖ Verify using **active buzzer** (has internal oscillator)
- ‚úÖ Check polarity (active buzzers have +/-)
- ‚úÖ Test with direct 5V connection
- ‚úÖ Verify BUZZER_PIN (GPIO 15)
- ‚úÖ Check Serial Monitor for buzzer activation logs
- ‚úÖ Test with manual toggle via web interface

### Settings Not Saving

**Problem**: Settings reset after reboot

**Solutions**:
- ‚úÖ Wait for "Settings saved" confirmation message
- ‚úÖ Check preferences.begin() in code
- ‚úÖ Verify flash memory isn't corrupted
- ‚úÖ Try erasing flash and re-uploading code
- ‚úÖ Check Serial Monitor for save errors
